# 과제 메모 — 컨텍스트 로트 관찰 (보너스 3번)

바이브 코딩 2주차 과제를 진행하며 `/context`를 열어 보고, 컨텍스트 로트
(반복·재발명·장황) 신호를 관찰한 기록입니다.

## 🔍 /context 확인

- `dragons.html`(드래곤 길들이기 인물·관계도) 작업 세션 시작 시점에 `/context`를 열어봄.
- 사용량: **약 31.4k / 1M 토큰 (3%)**.
  - 대부분 시스템 프롬프트 + 도구(MCP 등) 정의가 차지했고, 실제 대화 메시지는 매우 적었음.
  - → 세션 초반이라 아직 여유가 큰 상태였음.

## ⚠️ 목격한 컨텍스트 로트 신호

### 1. 반복 (repetition)
- **상황:** Claude in Chrome 브라우저 확장이 세션 내내 "연결 안 됨" 상태였는데,
  렌더링을 확인하려고 `tabs_context`(탭 조회)를 여러 번 **같은 방식으로 재시도**함.
- **왜 문제:** 같은 실패를 반복하는 것은 진전 없이 컨텍스트만 채우는 신호.
- **대응:** 반복을 멈추고 **기본 브라우저로 파일을 직접 여는 방식**으로 전환,
  검증은 `node --check`(JS 문법 검사) + 눈으로 확인으로 대체함.

### 2. 재발명 (re-invention)
- **상황:** 관계도 **범례(legend)** 렌더링 코드를 처음엔 `insertAdjacentText` 등을
  섞어 불필요하게 복잡하게 작성 → 곧바로 더 단순한 버전으로 **다시 작성**함.
- **왜 문제:** 첫 구현이 장황해지면 이후 수정·이해 비용이 커짐.
- **대응:** 커밋 전에 바로 알아채고 짧고 읽기 쉬운 코드로 리팩터링함.

### 3. 장황 (verbosity)
- **상황:** 기능을 여러 번 나눠 요청하다 보니(그림 추가 → 사람/드래곤 분리 →
  하트 효과 → 배경) 설명이 길어지는 경향이 있었음.
- **대응:** 매 단계 결과를 표·요약으로 압축해 전달하고, 다음 작업만 명확히 함.

## 💡 배운 점

- **신호가 보이면 같은 방법을 반복하지 말고 접근을 바꾼다.** (확장 미연결 → 대안 검증)
- **코드가 장황해지면 멈추고 단순화**한다. (범례 리팩터링)
- 긴 세션일수록 각 단계 결과를 **요약**해 컨텍스트를 정돈하면 로트를 줄일 수 있다.
